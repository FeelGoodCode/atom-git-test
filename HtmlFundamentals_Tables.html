HTML Fundamentals - Tables

Tables provide structure to data. Tables should not be used for formatting a page layout, but rather explictly used for dipslaying data that lends to being displayed in a table./p>

Overview

Tables are made of the several different HTML elements.

  • <caption> : Table title
  • <thead> : Single row, usually for column headers
  • <tr> : defines table row, required in header, body and footer
  • <th> : defines a cell for data in column header within an individual row in header.
  • <td> : defines a cell for data in the body or footer
  • <tfoot> : Single row usually for totals, summary, etc., comes before the body.
  • <tbody>: Multiple rows presenting data
Element Tags for Creating HTML Tables
HTML Tag Usage
Do not use tables to layout your document on the screen. It is better to use divs and css for page layouts and more complicated table structures.
<caption> The title of the table
<thead> Single row, usually for column headers
<tr> Defines table row, required in header, body and footer
<th> Defines a cell for data in column header within an individual row in header
<td> Defines a cell for data in the body or footer
<tfoot> Single row usually for totals, summary, etc., comes before the body
<tbody> Multiple rows representing data
<colspan> or <rowspan> Attributes used on table cells to span across columsn or rows

Formatting tables can ahppen with attributes such aside

  • width (align is deprecated)
  • border (not a lot of control with this)
  • cell padding - padding inside cell (works well with border not with frame and rules)
  • cell spacing - spacing between the cells (same note as cell padding)
  • no wrap (be careful with this one, use sparingly or not at all.)
  • frame and rules (use instead of border to give border and the vertical and horizontal rules separating tables and columns)
